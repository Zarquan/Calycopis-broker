    VolumeMount

    In Docker they are bind mounts of files or directories from the host filesystem.

    In our schema, volume mounts can point to a data resource OR a storage resource.

    In our entities data model, we could either have an abstract interface relationship, multiple targets, or specific types of mount.

    A single storage resource - as a directory.
        If path does not end with '/', path sets the full directory name.
        If path ends with '/', path sets the parent directory name.
        Resource name sets the target directory name.
        Individual data resources within the storage use resourceName.

    DataResourceVolume
    A single data resource - as a file.
        If path does not end with '/', path sets the full file name.
