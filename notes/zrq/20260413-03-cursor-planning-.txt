    >    │ Key Design Decisions                                                                                                                                                                     │
    >    │                                                                                                                                                                                          │
    >    │                                                                                                                                                                                          │
    >    │ 1. How to download HTTP data into the Docker volume                                                                                                                                      │
    >    │                                                                                                                                                                                          │
    >    │ There are two approaches:                                                                                                                                                                │
    >    │ • a) Run a helper container (e.g. alpine/curl or busybox) with the volume mounted, executing wget/curl to download the file. This works naturally with the container-in-container        │
    >    │   architecture since the Podman service on the host manages both the volume and the helper container.                                                                                    │
    >    │ • b) Use the Docker API to copy data (copyArchiveToContainerCmd). This is more complex and requires creating an intermediate tar archive.                                                │
    >    │                                                                                                                                                                                          │
    >    │ Recommendation: Approach (a) — run a lightweight helper container. It is simpler, works reliably across the Podman socket boundary, and avoids buffering the entire download in the      │
    >    │ broker's JVM memory.                                                                                                                                                                     │
    >    │                                                                                                                                                                                          │
    >    │ 2. Ordering: storage must be AVAILABLE before data can prepare                                                                                                                           │
    >    │                                                                                                                                                                                          │
    >    │ Currently PrepareSessionRequestEntity.beginPreparing() schedules all component prepare requests simultaneously. The DockerHttpResourceEntity prepare action needs the Docker volume to   │
    >    │  already exist (created by DockerVolumeMountStorageEntity's prepare action) before it can launch the download helper container.                                                          │
    >    │ Two options:                                                                                                                                                                             │
    >    │ • a) Use prepareStartInstant ordering — set a prepareStartInstant on the data resource that is slightly after the storage resource's expected prepare completion. This uses the          │
    >    │   existing scheduling mechanism.                                                                                                                                                         │
    >    │ • b) Check storage phase in data's prepare action — have DockerHttpResourceEntity.getPrepareAction() check whether its associated storage entity is AVAILABLE. If not, return a          │
    >    │   no-op and stay in PREPARING (the prepare loop will retry in 10 seconds).                                                                                                               │
    >    │                                                                                                                                                                                          │
    >    │ Recommendation: Approach (b) — it is simpler, more robust (no time estimation needed), and follows the existing pattern where the compute resource implicitly waits for its              │
    >    │ dependencies. The 10-second retry interval from DEFAULT_PREPARE_LOOP_INTERVAL is acceptable.                                                                                             │
    >    │                                                                                                                                                                                          │
    >    │ 3. Helper container image                                                                                                                                                                │
    >    │                                                                                                                                                                                          │
    >    │ A small image like alpine:3 with wget (included by default) is sufficient. The image name could be made configurable via application.yaml under broker.docker.helper-image for future    │
    >    │ flexibility.                                                                                                                                                                             │
    >    │                                                                                                                                                                                          │
    >    │ Implementation Steps                                                                                                                                                                     │
    >    │                                                                                                                                                                                          │
    >    │                                                                                                                                                                                          │
    >    │ Step 1: `DockerVolumeMountStorageEntity` — Create volume on prepare, delete on release                                                                                                   │
    >    │                                                                                                                                                                                          │
    >    │ In DockerVolumeMountStorageEntity.java:                                                                                                                                                  │
    >    │ • `getPrepareAction()`: Cast platform to DockerPlatform, get DockerClientFactory, call dockerClient.createVolumeCmd().exec(). Store the returned volume name in this.volumeIdent.        │
    >    │   Set phase to AVAILABLE.                                                                                                                                                                │
    >    │ • `getReleaseAction()`: Get the DockerClient, call dockerClient.removeVolumeCmd(this.volumeIdent).exec(). Set phase to COMPLETED.                                                        │
    >    │ • `getMonitorAction()`: Optionally inspect the volume; for now, transition straight to COMPLETED (existing behaviour is acceptable).                                                     │
    >    │ • Add @Column annotation to volumeIdent to ensure JPA persistence.                                                                                                                       │
    >    │                                                                                                                                                                                          │
    >    │ Access pattern (same as DockerSimpleComputeResourceEntity):                                                                                                                              │
    >    │                                                                                                                                                                                          │
    >    │ if (platform instanceof DockerPlatform) {                                                                                                                                                │
    >    │     DockerClientFactory clientFactory = ((DockerPlatform) platform).getDockerClientFactory();                                                                                            │
    >    │     try (DockerClient dockerClient = clientFactory.getDockerClient()) {                                                                                                                  │
    >    │         CreateVolumeResponse volume = dockerClient.createVolumeCmd().exec();                                                                                                             │
    >    │         this.volumeIdent = volume.getName();                                                                                                                                             │
    >    │     }                                                                                                                                                                                    │
    >    │ }                                                                                                                                                                                        │
    >    │                                                                                                                                                                                          │
    >    │                                                                                                                                                                                          │
    >    │ Step 2: `DockerHttpResourceEntity` — Download data into the volume via a helper container                                                                                                │
    >    │                                                                                                                                                                                          │
    >    │ In DockerHttpResourceEntity.java:                                                                                                                                                        │
    >    │ • `getPrepareAction()`:                                                                                                                                                                  │
    >    │   • Check that the associated storage entity is AVAILABLE. If not, remain in PREPARING (the processing loop will retry).                                                                 │
    >    │   • Get the DockerClient from the platform.                                                                                                                                              │
    >    │   • Get the volume name from this.getStorage() — since the storage is the DockerVolumeMountStorageEntity, its volumeIdent is now populated.                                              │
    >    │   • Get the data URL from this.getLocation().                                                                                                                                            │
    >    │   • Create and start a helper container:                                                                                                                                                 │
    >    │     • Image: alpine:3 (or configurable)                                                                                                                                                  │
    >    │     • Command: ["wget", "-O", "/data/output", "<http-url>"]                                                                                                                              │
    >    │     • Volume: Bind(volumeIdent, new Volume("/data"), AccessMode.rw)                                                                                                                      │
    >    │   • Wait for the helper container to exit.                                                                                                                                               │
    >    │   • Check exit code; if 0, set phase to AVAILABLE. If non-zero, fail with an error message.                                                                                              │
    >    │   • Remove the helper container.                                                                                                                                                         │
    >    │ • `getReleaseAction()`: No action needed (volume cleanup is handled by the storage entity).                                                                                              │
    >    │                                                                                                                                                                                          │
    >    │ The storage phase check can be done by navigating this.getStorage().getPhase():                                                                                                          │
    >    │                                                                                                                                                                                          │
    >    │ if (this.getStorage().getPhase() != IvoaLifecyclePhase.AVAILABLE) {                                                                                                                      │
    >    │     // Storage not ready yet, stay in PREPARING                                                                                                                                          │
    >    │     return new ComponentProcessingAction() {                                                                                                                                             │
    >    │         public void preProcess(LifecycleComponent c) {}                                                                                                                                  │
    >    │         public void process() {}                                                                                                                                                         │
    >    │         public void postProcess(LifecycleComponent c) {                                                                                                                                  │
    >    │             // Leave phase as PREPARING — the loop will retry                                                                                                                            │
    >    │         }                                                                                                                                                                                │
    >    │     };                                                                                                                                                                                   │
    >    │ }                                                                                                                                                                                        │
    >    │                                                                                                                                                                                          │
    >    │                                                                                                                                                                                          │
    >    │ Step 3: `DockerStorageLinker` — Already works for volumes                                                                                                                                │
    >    │                                                                                                                                                                                          │
    >    │ No changes needed. DockerVolumeMountStorageEntity.link() already calls setSourcePath(this.volumeIdent). Once volumeIdent is populated by Step 1, the linker bean will have all three     │
    >    │ parts (sourcePath, containerPath, accessMode) and isComplete() will return true. The DockerSimpleComputeResourceEntity will create the correct Bind object.                              │
    >    │                                                                                                                                                                                          │
    >    │ Step 4: Configuration (optional)                                                                                                                                                         │
    >    │                                                                                                                                                                                          │
