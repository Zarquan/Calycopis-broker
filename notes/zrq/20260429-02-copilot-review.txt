    Step 1 – Multi-module Maven structure

        Created java/calycopis-engine/pom.xml — no Spring, no calycopis-spring.
        Created java/calycopis-broker/pom.xml — depends on calycopis-engine + Spring Boot + calycopis-spring.

    OK, but might need more specific names.
    We are doing this so that we can support different frameworks and different front-ends.

    Paul wants to use Quarkus as the JPA framework.
    Paul wants to use an interface defined by VODML.

    Existing code base uses Spring Boot as the JPA framework.
    Existing code base uses an interface defined by OpenAPI.

    The business logic goes in calycopis-broker-engine.
    The foundation of the business logic is the broker functionality.
    We are splitting things into multiple jars, so need to keep track of where they come from.
    So everything that is a sub-project of calycopis-broker, starts with 'calycopis-broker'.

    Provide an interface only package calycopis-broker-api
    All of the components use these interfaces.
    The back-end business logic, both front-end packages, and the 3rd party components
    are all developed against these interfaces.

        calycopis-broker-api     - interface only, minimal beans

    calycopis-engine just contains some beans and interfaces
    calycopis-broker contains all of the entities

    PR comment :
    "This PR implements Steps 1–6 (and 7) of the multi-step plan to extract core business logic
    from the Spring Boot calycopis-broker into a new Maven module calycopis-engine that has zero
    dependencies on Spring, Spring Boot, or the net.ivoa.calycopis:calycopis-spring schema package."

    Yes, but .. calycopis-engine is not-functional.
    No business logic in it, no entities in it, just some empty shells.

    calycopis-broker contains all of the entities and business logic.
