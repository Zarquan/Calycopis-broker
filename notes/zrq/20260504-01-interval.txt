    Reasons for removing intervals in the schema ..
    Because String parsing is fragile.

    String parsing in Java is pretty robust.
    Using the Interval, Instant and Duration classes.

    One of the reasons was to bring the schema closer to the database.
    So that generating JPA entities from the VODML would create the right things.

        available:
          start:
            ISO8601Interval
          duration:
            ISO8601Duration

        available:
          start:
            start: long
            range: long
          duration:
            long

    BUT .. it looks like we probably aren't going to generate core JPA entities from the VODML.
    so why do we need this now ?
